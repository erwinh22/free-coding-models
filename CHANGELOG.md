## 0.1.67

### Added

- **Stability Score** ‚Äî new composite 0‚Äì100 metric combining p95 latency (30%), jitter/œÉ (30%), spike rate (20%), and uptime (20%). Displayed as a "Stab" column in the TUI with color-coded thresholds (green ‚â•80, cyan ‚â•60, yellow ‚â•40, red <40).
- **p95 latency** (`getP95`) ‚Äî 95th percentile latency from successful pings. Answers "95% of requests are faster than X ms."
- **Jitter** (`getJitter`) ‚Äî standard deviation of latency. Low jitter = predictable, high jitter = erratic/spiky.
- **"Spiky" verdict** ‚Äî new verdict that catches models with good average latency but terrible tail latency (p95 spikes). A model with avg 250ms but p95 6000ms now gets flagged as "üìà Spiky" instead of "üöÄ Perfect".
- **Stability sorting** ‚Äî press `B` to sort by stability score. Most stable models rise to the top.
- 24 new unit tests covering p95, jitter, stability score, Spiky verdict, and stability sorting.

### Changed

- `getVerdict()` is now stability-aware: models in "Perfect" or "Normal" avg range get downgraded to "Spiky" when p95 shows extreme tail latency (requires ‚â•3 pings to avoid false positives).
- `findBestModel()` now uses a 4-key sort: status ‚Üí avg latency ‚Üí stability score ‚Üí uptime (was 3-key: status ‚Üí avg ‚Üí uptime).
- `sortResults()` supports new `'stability'` column.
- `VERDICT_ORDER` updated to include "Spiky" between "Slow" and "Very Slow".

## 0.1.4 ‚Äî 2026-02-22

### Fixed
- **OpenClaw config structure**: `providers` was incorrectly written at the config root. Moved to `models.providers` per official OpenClaw docs (`docs.openclaw.ai/providers/nvidia`).
- **OpenClaw API key storage**: Removed `apiKey` from provider block (not a recognized field). API key is now stored under `env.NVIDIA_API_KEY` in the config.
- **OpenClaw models array**: Removed the `models: []` array from the provider block (OpenCode format, not valid in OpenClaw).
- **`openclaw restart` CLI command doesn't exist**: Replaced hint with correct commands ‚Äî `openclaw models set` / `openclaw configure`. Gateway auto-reloads on config file changes.
- **OpenClaw model not allowed**: Model must be explicitly listed in `agents.defaults.models` allowlist ‚Äî without this, OpenClaw rejects the model with "not allowed" even when set as primary.
- **README**: Updated OpenClaw integration section with correct JSON structure and correct CLI commands.
